# Phase 2 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Surfaces in scope (5):**
1. Push gate — full-screen takeover replacing `Gate.tsx`'s Phase 1 bypass shell: custom pre-prompt, iOS Add-to-Home-Screen walkthrough, decline/re-ask (PUSH-01…05, ID-04)
2. OS push notification content — native browser/OS notification only, no custom in-app UI (PUSH-06, PUSH-07, D-13, D-14)
3. Owner reply composer draft-preview — draft (own language) → translate-preview (visitor's language, editable) → send, in `ReplyBox.tsx` (TRANS-02, TRANS-03, D-09, D-10, D-11)
4. Message bubble "show original" toggle — both directions: visitor sees owner replies translated with tap-to-reveal original (`MessageBubble.tsx`, D-12), owner sees visitor messages translated with tap-to-reveal original (`Thread.tsx`, ADMIN-09)
5. Admin conversation list additions — gate funnel stats row + per-row "unreachable" badge, in `page.tsx` / `ConversationRow.tsx` (OPS-11, D-17, D-18, D-19)

**Explicitly NOT in scope:** the real prioritized inbox, filters, search, at-a-glance counts, status/faith controls, block/delete, crisis resources, admin lockout, the presence *toggle* (all Phase 3). No retry/re-notify action on the unreachable badge (D-19 — informational only).

## Design System

Inherited unchanged from Phase 1 (`01-UI-SPEC.md`, approved 2026-07-20). shadcn is already initialized — no re-init, no new questions asked.

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | Already resolved in `components.json`: `style: radix-nova`, `baseColor: neutral`, `cssVariables: true`, `rtl: true` |
| Component library | radix (via shadcn) |
| Icon library | `lucide-react` |
| Font | System stack — no webfont, no CDN (unchanged; see 01-UI-SPEC.md rationale — script coverage, privacy/ID-05, no-FOUT) |

**No new shadcn components are required this phase.** Existing installs (`button`, `drawer`, `input`, `label`, `textarea`) cover every new surface:
- The push gate is a full-viewport replacement of chat content (same pattern `Gate.tsx` already uses to swap in/out of `children`), not a `dialog`/`drawer` overlay — it needs no new primitive.
- The "show original" toggle is inline expand/collapse text, not a sheet.
- The gate-funnel stats row and the unreachable badge are plain text, matching Phase 1's registry-minimalism call (badge, dialog, dropdown-menu, select, toast, card, avatar, tabs remain deliberately uninstalled).

### Known color discrepancy — carried forward, not in scope to fix

`01-UI-SPEC.md` specified a teal accent (`hsl(173 58% 27%)` light / `hsl(172 46% 52%)` dark). The shipped `src/app/globals.css` instead carries shadcn's raw `neutral` defaults (`--primary: oklch(0.205 0 0)` light / `oklch(0.922 0 0)` dark — near-black / near-white grayscale), and every Phase 1 component (`MessageBubble`, `ReplyBox`, `ConversationRow`) already renders against these actual tokens. This phase inherits the **actual shipped tokens**, documented below under Color, not the aspirational spec — re-theming `globals.css` is out of scope for Phase 2. Flag this to the owner only if they ask why bubbles look grayscale instead of teal.

## Spacing Scale

Unchanged from Phase 1 — all multiples of 4, no new tokens introduced.

| Token | Value | Tailwind | Phase 2 usage |
|-------|-------|----------|----------------|
| xs | 4px | `1` | Gap between "See original" toggle and its chevron |
| sm | 8px | `2` | Gap between composer's translated textarea and its "See original"/"Edit original" action row |
| md | 16px | `4` | Default: gate page gutter, stats-row internal padding, unreachable-badge inline gap from timestamp |
| lg | 24px | `6` | Gap between gate heading/body/CTA blocks |
| xl | 32px | `8` | Top inset above the gate heading; top/bottom inset around the iOS walkthrough animation container |
| 2xl | 48px | `12` | Reserved — no Phase 2 usage |
| 3xl | 64px | `16` | Reserved — no Phase 2 usage |

**Touch targets:** unchanged floor of `44×44px` (`size-11`) — applies to the gate's primary CTA, the walkthrough's "I've added it" CTA, the re-ask "Try again" CTA, the composer's "Edit original" action, and the "See original" toggle (even though it's Label-sized text, its tappable hit area must still clear 44px via padding, not font size).

**iOS walkthrough animation container:** fixed aspect ratio `4:5` (portrait — matches an iPhone screenshot's proportions), max-width `320px`, centered. Captions render as separate DOM text below the container (never burned into the asset) so they can be localized without re-exporting the animation.

Exceptions: none.

## Typography

Unchanged three-size, two-weight system from Phase 1. No fourth size is introduced.

| Role | Size | Weight | Line Height | Phase 2 usage |
|------|------|--------|-------------|----------------|
| Body | 16px | 400 | 1.5 | Gate explanation body, re-ask body, iOS walkthrough intro + step captions, composer draft/preview textarea text (unchanged from Phase 1's composer role) |
| Label | 14px | 400 | 1.4 | Gate CTA aria-labels' visible fallback text (if any), "Translating…"/"Confirming…" transient states, "See original"/"Hide original"/"Edit original" toggles, admin stats-row numbers and labels, unreachable badge text |
| Heading | 20px | 600 | 1.3 | Gate heading, re-ask heading, iOS walkthrough heading |
| Display | — | — | — | **Not used. Do not introduce a fourth size.** |

Weight 600 stays reserved for the Heading role only — the gate is a full-screen moment but is not an exception to "nothing in a conversation, or its surrounding chrome, is bolded for emphasis beyond headings."

## Color

Base: shadcn `neutral`, as actually shipped in `src/app/globals.css` (see discrepancy note above — this table documents shipped values, not 01-UI-SPEC.md's original teal spec).

| Role | Light | Dark | Usage |
|------|-------|------|-------|
| Dominant (60%) | `oklch(1 0 0)` | `oklch(0.145 0 0)` | `--background` — gate screen background (same token as chat, no separate onboarding palette) |
| Secondary (30%) | `oklch(0.97 0 0)` | `oklch(0.269 0 0)` | `--muted` — composer bar (unchanged), the "See original" expanded block's background |
| Accent (10%) | `oklch(0.205 0 0)` | `oklch(0.922 0 0)` | `--primary` — see reserved list below |
| Destructive | `oklch(0.577 0.245 27.325)` | `oklch(0.704 0.191 22.216)` | Translation-preview failure text only (new); failed-message state (unchanged) |
| Muted foreground | `oklch(0.556 0 0)` | `oklch(0.708 0 0)` | Stats-row labels, unreachable badge, transient "Translating…"/"Confirming…" text, walkthrough captions |

**Accent (`--primary`) reserved list — extends Phase 1's three items with exactly two more, same discipline ("one primary action per screen, everything else neutral"):**

1. *(unchanged)* The visitor's own outgoing message bubble background
2. *(unchanged)* The send button, only while the composer is non-empty
3. *(unchanged)* The focus ring on composer/language-sheet/inputs
4. **New:** The push gate's single primary CTA per screen — "Allow notifications" (pre-prompt), "I've added it" (iOS walkthrough), "Try again" (re-ask). Exactly one accent-colored button is visible on the gate at any time; there is no secondary "skip" button to compete with it (D-06's softening happens automatically on the visitor's *next* visit, not via an in-page control this visit).
5. **New:** The owner composer's Preview/Send button in its **Send** state (its **Preview** state — see Copywriting Contract — reuses this exact button, unchanged styling, just a different label/aria).

Nothing else gets accent. Not the "See original" toggle (Label/muted-foreground, matching Phase 1's low-weight-affordance precedent), not the stats row, not the unreachable badge, not the iOS walkthrough's step indicators.

**Destructive is used for exactly one new thing this phase:** the owner composer's "Couldn't translate" fallback notice. It is not used for the gate's decline/re-ask state — declining push is not an error, and a red re-ask would contradict D-06/D-03's explicitly gentle, non-alarming tone (same principle as Phase 1's rate-limit copy never using destructive framing).

## Copywriting Contract

Two copy domains this phase, with different localization rules — **read this before writing any string:**

- **Visitor-facing copy is a locale JSON key, localized across all ten languages** (LANG-07): gate heading/body/CTA, re-ask heading/body/CTA, iOS walkthrough heading/intro/captions/CTA, the "See original"/"Hide original" toggle on owner messages, and the push notification title/body.
- **Admin-facing copy is authored directly in the component as English source text, exactly like Phase 1's `ReplyBox.tsx`/`ConversationRow.tsx`/`page.tsx` precedent** — the owner is one fixed person; ADMIN-side UI is never localized. This covers: the composer's "Preview"/"Translating…"/"See original"/"Hide original"/"Edit original"/"Couldn't translate…" strings, the stats-row labels, and the unreachable badge text.

| Element | Copy | Locale key / location |
|---------|------|------------------------|
| Gate heading | **"Stay reachable"** | `pushGateHeading` |
| Gate body | **"So I can reach you after you close this tab, allow notifications. You'll only ever be told a reply is waiting — nothing about it is shown until you open the chat yourself."** | `pushGateBody` |
| Gate primary CTA | **"Allow notifications"** | `pushGateAllowCta` |
| Gate confirming (transient, post-grant probe) | **"Confirming…"** | `pushGateConfirming` |
| Re-ask heading | **"Without notifications, I can't reach you again"** | `pushGateDeclinedHeading` |
| Re-ask body | **"If you leave now, you can still open this chat later, but I won't be able to let you know when I've replied. You can still say yes."** | `pushGateDeclinedBody` |
| Re-ask retry CTA | **"Try again"** | `pushGateRetryCta` |
| iOS walkthrough heading | **"Add One Chat to your Home Screen first"** | `iosWalkthroughHeading` |
| iOS walkthrough intro | **"iPhone only allows notifications for apps added to your Home Screen. It takes a few seconds."** | `iosWalkthroughIntro` |
| iOS walkthrough step 1 caption | **"Tap the Share icon"** | `iosWalkthroughStep1` |
| iOS walkthrough step 2 caption | **"Tap \"Add to Home Screen\""** | `iosWalkthroughStep2` |
| iOS walkthrough step 3 caption | **"Open One Chat from your Home Screen to continue"** | `iosWalkthroughStep3` |
| iOS walkthrough CTA | **"I've added it"** | `iosWalkthroughCta` |
| Visitor "show original" (owner messages) | **"See original"** / **"Hide original"** | `showOriginalLabel` / `hideOriginalLabel` |
| Push notification title | **"You have a new reply"** | `pushNotificationTitle` |
| Push notification body | **"Tap to open the conversation."** | `pushNotificationBody` |
| Owner composer preview CTA (aria-label) | **"Preview translation"** | Component-local (English, admin) |
| Owner composer translating (transient) | **"Translating…"** | Component-local (English, admin) |
| Owner composer send CTA (preview state, aria-label) | **"Send"** (reuses existing `ReplyBox` send button, unchanged) | Component-local (English, admin) |
| Owner "see original" toggle | **"See original"** / **"Hide original"** | Component-local (English, admin) |
| Owner "edit original" action (aria-label) | **"Edit your original message"** | Component-local (English, admin) |
| Owner composer translation-failed notice | **"Couldn't translate. You can still send your original message."** | Component-local (English, admin) |
| Owner composer "send anyway" CTA | **"Send anyway"** | Component-local (English, admin) |
| Admin unreachable badge | **"Unreachable"** (aria-label: **"This visitor's push notifications are no longer working"**) | Component-local (English, admin) |
| Admin stats-row labels | **"Shown"**, **"Reached prompt"**, **"Granted"**, split by **"iOS"** / **"Other"** | Component-local (English, admin) |
| Destructive confirmation | **None in this phase.** Block and delete are Phase 3. | — |

**Copy rules binding on the executor:**

- **The gate has exactly one visible primary CTA at any time** — never two competing buttons on the same screen (no in-page "skip" button; D-06's softening is a *next-visit* behavior, not an in-page control).
- **The re-ask is gentle, never alarming** — no destructive color, no countdown, no "you have been blocked" framing. Same tone discipline as Phase 1's rate-limit copy.
- **"Translating…" and "Confirming…" are bounded, never indefinite** — both must resolve (success or fallback) within a short timeout; neither is a spinner icon, both are plain Label-role text replacing the button's normal label, matching the house rule against indefinite spinners.
- **The composer's translated text is directly editable** (D-10) — no separate "approve/reject" step. The only safety net is the visitor's own "See original" affordance on their end (D-12), not a UI-level guard on the owner's edit.
- **"See original" never renders for same-language message pairs** (TRANS-06) — if sender and reader share a language, only the single original bubble renders; there is no dead/disabled toggle.
- **Push notification copy is fixed per language, authored once into locale JSON — never translated live at send time** (D-14). No visitor-specific content (name, preview, faith reference) ever appears in it (PUSH-07).
- **The unreachable badge carries no action** (D-19) — no retry, no re-notify button. Purely informational.
- **Admin-side new strings are English-only, hardcoded in the component** — do not create locale JSON keys for the composer's preview UI, the stats row, or the unreachable badge; this matches Phase 1's own admin-copy precedent (`ReplyBox.tsx`'s "Write a reply"/"Send" are hardcoded, not localized).

Applicable state considerations resolved: 10 covered, 2 backstop, 2 unresolved

| Category | Element(s) | Status | Resolution / Reason |
|----------|------------|--------|---------------------|
| loading | gate-permission-request | ✅ covered | `requestPermission()` fires synchronously as the CTA handler's first statement (PUSH-04); no custom loading UI precedes the OS dialog — the OS dialog itself is the only visible "loading" state |
| loading | gate-probe-confirming | ✅ covered | After grant, the gate shows the bounded "Confirming…" Label text (never a spinner) while the round-trip probe (PUSH-12) runs, then resolves to chat |
| loading | translation-preview | ✅ covered | The composer shows the bounded "Translating…" Label text in place of the send icon while the preview request is in flight; never an indefinite spinner (mirrors TRANS-09's bound on the visitor side) |
| error | translation-preview-failed | ✅ covered | On failure/timeout the composer shows "Couldn't translate. You can still send your original message." with a "Send anyway" CTA that sends the untranslated original (D-11/TRANS-03) |
| error | push-permission-denied-or-dismissed | ✅ covered | A denied/dismissed prompt shows the gentle re-ask screen (PUSH-03); no destructive color, no countdown — same tone discipline as Phase 1's rate-limit copy |
| error | gate-probe-failed | ⚠ unresolved | No copy was specified for a round-trip probe (PUSH-12) failing after permission was already granted; planner assumption: let the visitor into chat regardless (permission can't be un-granted programmatically), log server-side only, no user-facing error |
| populated | admin-unreachable-badge | ✅ covered | A conversation whose subscription 404/410'd renders the quiet inline "Unreachable" label next to its row; purely informational, no action (D-18/D-19) |
| populated | admin-gate-funnel-stats-row | ✅ covered | All-time totals (Shown / Reached prompt / Granted) render as one small stats row split by iOS / Other |
| empty | admin-gate-funnel-stats-zero-state | ✅ covered | Before any visitor has hit the real gate, all six counts render as literal "0" rather than the row being hidden or showing a placeholder dash |
| zero-one-many | admin-gate-funnel-stats-formatting | ⚠ unresolved | Large-count formatting (1,000+) wasn't specified; planner assumption: plain `formatDigits()`-run integers, no locale thousand-separators — consistent with Phase 1's ASCII-digit-everywhere rule |
| partial | show-original-toggle | ✅ covered | Only messages that were actually translated (sender's language differs from the reader's) render the toggle at all; same-language pairs (TRANS-06) show only the single original bubble with no dead toggle |
| partial | owner-composer-edit-original | ✅ covered | Tapping "Edit original" discards the current preview and returns the composer to draft mode pre-loaded with the original text — it never silently merges an edited translation back onto the stored original |
| overflow | ios-walkthrough-animation | ✅ covered | The animation container is a fixed 4:5 aspect-ratio box that never grows with content; captions live outside it as separate DOM text, never burned into the asset, so they localize without re-exporting the animation |
| long-text | push-gate-copy | 🧪 backstop | Gate heading/body and re-ask copy must clear the fold at 360×640 in every one of the ten languages, same backstop class as Phase 1's welcome-all-locales row |
| long-text | ios-walkthrough-captions | 🧪 backstop | Step captions must wrap without overlapping the animation container at 360×640 in the longest-expanding languages (Arabic, Hindi, Russian) |
