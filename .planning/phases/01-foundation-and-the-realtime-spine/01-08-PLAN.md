<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Anonymous visitor -> POST /api/chat/messages | Fully untrusted input on every byte — no auth beyond the signed cookie's identity claim |
| pg_notify payload | Must never carry message content — 8000-byte limit and the no-pastoral-content-in-logs requirement both depend on this |
| Rate-limit key derivation | IP is hashed inline in this route; the raw IP must never reach a log line, a DB column, or the pg_notify payload |

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-23 | Tampering | src/app/api/chat/messages/route.ts | high | mitigate | zod schema validates every field before any DB write; Drizzle/postgres.js parameterized queries throughout — no string-concatenated SQL |
| T-01-24 | Information Disclosure | pg_notify payload | high | mitigate | Payload is `{c, m, k}` pointers only, constructed explicitly without the body field in scope — enforced by the acceptance criteria's negative grep |
