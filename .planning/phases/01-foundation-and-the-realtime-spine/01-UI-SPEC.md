# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Surfaces in scope (4):**
1. Visitor chat shell — full-screen, single conversation (CHAT-01…09, LANG-01…07)
2. Language bottom sheet — 10 endonyms (D-09)
3. Owner login + one-time setup page (ADMIN-01, D-14)
4. Owner flat conversation list + thread (ADMIN-03, D-12)

**Explicitly NOT in scope:** the real push `<Gate>` (Phase 2 — a shell behind an env bypass flag here, no designed UI), translation UI (Phase 2), presence toggle, prioritized inbox, filters, search, counts, status/faith controls, block/delete, crisis resources (Phase 3).

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn |
| Preset | `npx shadcn init --template next --base radix --css-variables --rtl -y` — record the resolved preset string back into `components.json` |
| Component library | radix (via shadcn `--base radix`) |
| Icon library | `lucide-react` |
| Font | System stack — no webfont, no CDN |

**shadcn initialization is the first UI task of this phase.** Tailwind v4 has no `tailwind.config.*`; theming lives entirely in `app/globals.css` via `@theme inline`. `components.json` must carry `"tailwind": { "config": "", "baseColor": "neutral", "cssVariables": true }`.

**`--rtl` is load-bearing, not cosmetic.** shadcn's shared `shadcn/tailwind.css` carries the RTL fixes for Radix primitives. Initializing without it means hand-patching every popover/drawer side. LANG-03 depends on this flag.

### Font decision — system stack, no webfont

```css
--font-sans: ui-sans-serif, system-ui, -apple-system, "Segoe UI", Roboto,
  "Helvetica Neue", "Noto Sans", "Noto Sans Arabic", "Noto Sans Devanagari",
  Arial, sans-serif, "Apple Color Emoji", "Segoe UI Emoji";
```

Three binding reasons, all specific to this product:

1. **Script coverage.** Phase 1 ships up to ten languages spanning Arabic, Devanagari, Han, Cyrillic and Latin. Self-hosting Noto across all five is multiple megabytes to a mobile-first audience. Every OS already ships a correctly-hinted font for its own locale.
2. **Privacy.** A Google Fonts CDN request discloses the visitor's IP to a third party on a product whose hard requirement is that no personal data is collected (ID-05). A system stack makes zero network requests.
3. **No FOUT.** D-11 forbids a flash of wrong theme or direction. A webfont reintroduces a flash of unstyled text on exactly the first paint that decision protects.

Do not add `next/font`. Do not add a display font.

### Icon mirroring allowlist (LANG-05)

Directional icons mirror under `dir="rtl"`; everything else must not. Implement as an explicit allowlist class (`rtl:-scale-x-100`), never a blanket rule.

| Icon | Phase 1 usage | Mirrors in RTL |
|------|---------------|----------------|
| `ArrowUp` / `SendHorizontal` | Composer send | **Yes** |
| `ChevronLeft` | Admin thread back | **Yes** |
| `RotateCcw` | Failed-message retry | **Yes** |
| `Languages` | Header language control | No |
| `Sun` / `Moon` | Header appearance control | No |
| `Check` | `sent` delivery state | No |
| `X` | Close language sheet | No |
| Timestamps, digits, message IDs | Everywhere | No — never mirrored, never bidi-reordered |

## Spacing Scale

Declared values (all multiples of 4 — maps 1:1 onto the Tailwind v4 default `0.25rem` unit, so **no custom spacing tokens are defined**):

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `1` | Icon-to-label gap, delivery-state icon inset |
| sm | 8px | `2` | Gap between consecutive bubbles from the same author |
| md | 16px | `4` | Default: bubble inner padding, composer padding, horizontal page gutter |
| lg | 24px | `6` | Gap between author turns, welcome block bottom margin |
| xl | 32px | `8` | Header height padding, language-sheet top padding |
| 2xl | 48px | `12` | Empty-state vertical inset (admin list) |
| 3xl | 64px | `16` | Reserved — no Phase 1 usage |

**Touch targets:** every interactive element is minimum `44×44px` (`size-11`). This is a multiple of 4 and therefore not an exception — it is a floor, applied to the two header controls, the send button, each language-sheet row, and the tap-to-retry affordance. Most visitors arrive one-handed on mobile.

**Composer safe area:** the composer bar adds `pb-[env(safe-area-inset-bottom)]` on top of its `md` padding. This is a device inset, not a spacing token.

Exceptions: none.

## Typography

Exactly three sizes, exactly two weights.

| Role | Size | Weight | Line Height | Applied to |
|------|------|--------|-------------|------------|
| Body | 16px | 400 | 1.5 | Message text, composer textarea, language endonyms, admin thread text |
| Label | 14px | 400 | 1.4 | Timestamps, presence status line, failed-state text, admin list preview, form labels |
| Heading | 20px | 600 | 1.3 | Welcome line 1, admin screen titles, language-sheet title |
| Display | — | — | — | **Not used in Phase 1. Do not introduce a fourth size.** |

**16px body is load-bearing, not a preference.** iOS Safari zooms the viewport when a focused input renders below 16px. The composer is the single most-used control in the product; sub-16px there breaks the layout on every send.

**Heading line-height is 1.3, not the usual 1.2.** Arabic diacritics and Devanagari conjuncts occupy more vertical space than Latin ascender/descender pairs; 1.2 clips them. 1.3 is the smallest value safe across all ten scripts.

Weight 600 is reserved for the Heading role only. Message text, sender attribution, and delivery states are all 400 — nothing in a conversation is bolded for emphasis.

## Color

Base: shadcn `neutral`. Both light and dark are first-class (LANG-06); neither is the "real" theme.

| Role | Light | Dark | Usage |
|------|-------|------|-------|
| Dominant (60%) | `hsl(0 0% 100%)` | `hsl(0 0% 3.9%)` | `--background` — the chat canvas, admin page background |
| Secondary (30%) | `hsl(0 0% 96.1%)` | `hsl(0 0% 14.9%)` | `--muted` — owner (incoming) bubbles, composer bar, language-sheet surface, admin list rows |
| Accent (10%) | `hsl(173 58% 27%)` | `hsl(172 46% 52%)` | `--primary` — see reserved list below |
| Destructive | `hsl(0 72% 41%)` | `hsl(0 63% 55%)` | Failed-message state only |
| Muted foreground | `hsl(0 0% 45.1%)` | `hsl(0 0% 63.9%)` | Timestamps, presence status line, delivery-state icon |

Accent (`--primary`) is reserved for **exactly three things**:

1. The **visitor's own outgoing message bubble** background (foreground: `--primary-foreground`)
2. The **send button**, and only while the composer is non-empty (empty composer → `--muted-foreground` icon on transparent)
3. The **focus ring** on the composer and on language-sheet rows

Nothing else. Not the language control, not the appearance toggle, not the welcome, not the presence line, not the admin list, not links. The outgoing bubble consumes the entire accent budget in an active conversation — that is why the chrome gets none of it.

**Accent contrast:** light `hsl(173 58% 27%)` against white foreground ≈ 7.2:1; dark `hsl(172 46% 52%)` against `hsl(0 0% 3.9%)` foreground ≈ 8:1. Both clear WCAG AA for body text with headroom.

**The presence status line has no color and no dot** (D-06). It is `--muted-foreground` Label text under the header, nothing more. A colored dot is exactly the support-widget signal the product rejects — and a green/grey dot would read as a third header control against CHAT-09's "exactly two."

**Destructive is used once in Phase 1** — the failed-message state. There are no destructive *actions* in this phase (block and delete are Phase 3), so no destructive button and no confirmation dialog exists yet.

## Copywriting Contract

Authored in English as the source locale, then localized to the confirmed list per LANG-07. Every string below is a key in the locale JSON — none may be hardcoded in a component.

| Element | Copy |
|---------|------|
| Primary CTA | Send button — icon-only, `aria-label`: **"Send message"** |
| Secondary control 1 | Language control — `aria-label`: **"Change language"**; sheet title: **"Choose your language"** |
| Secondary control 2 | Appearance control — `aria-label`: **"Switch to dark appearance"** / **"Switch to light appearance"** |
| Composer placeholder | **"Write a message"** |
| Welcome line 1 (warmth) | **"You're welcome here. Whatever's on your mind, you can say it."** |
| Welcome line 2 — owner online | **"I'm here now."** |
| Welcome line 2 — owner offline | **"I'm not at my desk right now, but I read every message and I will reply."** |
| Empty state heading (admin list) | **"No conversations yet"** |
| Empty state body (admin list) | **"When someone opens the chat and sends a message, it will appear here."** |
| Error — send failed | **"Couldn't send. Tap to try again."** |
| Error — rate limited | **"You're sending faster than I can keep up. Give it a moment — nothing you wrote is lost."** |
| Error — reconnect stuck | **"Reconnecting…"** |
| Error — admin login | **"That email or password isn't right."** |
| Admin CTA — login | **"Sign in"** |
| Admin CTA — setup | **"Create owner account"** |
| Destructive confirmation | **None in this phase.** Block and delete are Phase 3. |

**Copy rules binding on the executor:**

- **The visitor chat has no empty state.** The welcome renders client-side from locale JSON (D-05) and is never a row in `messages`. It re-renders instantly on language switch. The chat is never a blank canvas.
- **The welcome is two lines and only two lines** (D-08). No third line, no "who I am" preamble, no anonymity explainer. It must clear the fold at 360×640 in every script in the final list.
- **Line 2 is chosen by live presence** (D-06/D-07) and swaps over the open SSE stream. It must never claim the owner is present when they are not.
- **Failure copy is attached to one bubble, never to the screen.** On send failure the only thing that changes is that specific bubble (D-18). No toast, no banner, no dialog. The visitor's typed text is never removed from the screen for any reason (D-19) — *someone who just typed something hard to say must never have to type it twice.*
- **Reconnect is silent by default** (D-17). The ~4-minute server-side stream recycle (D-15) is normal operation and produces zero UI. "Reconnecting…" appears only after repeated consecutive failures, as one quiet Label line — never a spinner, never a modal.
- **Rate-limit copy is gentle, never a cutoff** — *someone in crisis typing fast must not be stonewalled.* No countdown, no "you have been blocked", no lockout language.
- **No "delivered", no "seen", ever** (D-20). Exactly two delivery states exist: `sent` (a `Check` at `--muted-foreground`, 14px) and `failed`. Optimistic bubbles render at 60% opacity until durable persistence is confirmed, then settle to 100%.
- **No typing indicator.** Out of scope by product decision, not by phase.
- **Admin login must not enumerate.** One string for wrong email and wrong password alike.

| Category | Element(s) | Status | Resolution / Reason |
|----------|------------|--------|---------------------|
| empty | visitor-chat | ✅ covered | A first-visit chat renders the two-line client-side welcome from locale JSON; the canvas is never blank and nothing is inserted into `messages` |
| empty | admin-conversation-list | ✅ covered | Zero conversations render the documented "No conversations yet" heading and body copy |
| loading | first-paint-theme-direction | ✅ covered | `<html lang dir>` and the theme class are correct in the first byte from the server-set cookie; the pre-paint script only corrects a missing-cookie/present-localStorage case, so no flash of wrong theme or wrong direction occurs |

| loading | sse-reconnect | ✅ covered | The ~4-minute stream recycle and its reconnect produce no UI at all; "Reconnecting…" renders only after repeated consecutive failures |
| error | message-send-failure | ✅ covered | After automatic retries are exhausted, only that one bubble changes to the failed state with tap-to-retry; the visitor's typed text is never removed from the screen |
| error | rate-limited-send | ✅ covered | A rate-limited send renders the gentle localized line with no countdown, no lockout copy, and no loss of typed text |
| error | admin-login-failure | ✅ covered | Wrong email and wrong password render one identical string, so the form does not enumerate accounts |
| populated | long-conversation-scroll | ✅ covered | The thread anchors to the newest message on open and on a new message *only when the visitor is already at the bottom*; it never yanks the view away from someone scrolled up reading history |
| partial | mixed-direction-message | ✅ covered | Every message body carries `dir="auto"` with `unicode-bidi: isolate`, so a Latin URL or a scripture reference inside Arabic text renders intact and unmangled (LANG-04) |
| overflow | composer-growth | ✅ covered | The composer textarea grows to a maximum of 5 lines, then scrolls internally; the send button and the message list never leave the viewport |
| long-text | message-bubble | 🧪 backstop | A single unbroken 500-character token and a 200-character URL must wrap inside the bubble without horizontal page scroll, in both `dir=ltr` and `dir=rtl` |
| long-text | welcome-all-locales | 🧪 backstop | Both welcome lines must clear the fold at 360×640 in Arabic, Hindi, Chinese, Russian and Spanish — the longest expansions in the list |
| long-text | admin-list-preview | 🧪 backstop | The last-message preview truncates to a single line with logical-property ellipsis, correct at both the inline-start and inline-end under RTL |
| overflow | language-sheet | 🧪 backstop | Ten endonym rows in a bottom sheet capped at `70vh` scroll internally on a 360×640 viewport with every row still meeting the 44px target |
| zero-one-many | admin-conversation-list | ⚠ unresolved | Zero and one are specified; the many-case scale ceiling is unknown at Phase 1 (flat list, no filters per D-12). Planner treats "plain scroll without virtualization is adequate" as an assumption to revisit in Phase 3 |

## Registry Safety

| Registry | Blocks Used | Safety Gate |
|----------|-------------|-------------|
| shadcn official | `button`, `drawer`, `input`, `label`, `textarea` | not required |

**No third-party registries are declared for this phase.** Any addition requires `npx shadcn view {block} --registry {url}` and a recorded review before it enters this contract.

`drawer` pulls `vaul` (built on Radix Dialog) — this is the bottom-sheet primitive for the language picker (D-09) and is shipped by the official registry.

Components deliberately **not** installed in Phase 1, to keep the surface honest: `dialog`, `dropdown-menu`, `select`, `toast`/`sonner`, `card`, `badge`, `avatar`, `tabs`. The header has exactly two controls (CHAT-09), failure surfaces on the bubble rather than in a toast, and the admin list is a flat list of rows — none of these earn a component yet.
