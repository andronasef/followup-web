<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> PATCH /api/chat/prefs | Unauthenticated-by-design (any visitor can change their own lang/appearance) but scoped strictly to the calling visitor's own cookie — never accepts a visitor id as a body parameter |
| localStorage <-> signed cookie | localStorage is script-writable/XSS-exfiltratable; the cookie is httpOnly and immune to that — the design deliberately never trusts localStorage for identity, only for display recovery |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-16 | Spoofing | src/app/api/chat/prefs/route.ts | high | mitigate | The route derives the visitor identity exclusively from the verified httpOnly cookie via requireVisitor() — it never accepts a visitor id in the request body, so a caller cannot alter another visitor's preferences |
| T-01-17 | Tampering | localStorage mirror | medium | accept | localStorage is inherently script-writable; accepted because it is never used as an auth input (cookie wins on conflict per D-11) — a tampered localStorage value only affects the visitor's own pre-paint display guess, never their identity or another visitor's data |
| T-01-18 | Information Disclosure | Accept-Language header | low | accept | Standard browser-sent header, used only to select a key into a fixed language enum; no sensitive inference is drawn from it |
</threat_model>
